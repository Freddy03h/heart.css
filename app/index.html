Heart.css

The Heart.css framework is a micro-framework to describe the Layout of a Single Page Application.

It built for design an Application instead of a Document and it not provide graphic or typograhpic style like Bootstrap or Fundation. This framework was build with the Mobile-first approach and it's completely Fluid. So you can easily use it to build your Responsive Web Design Application. It takes a lot use of position: absolute for cancel many of browser's behavior that are presume it run a Document.

Heart.css have been tested on Chrome Mac and Android, Firefox Mac and Android, Safari Mac and iOS, Android 4.X default browser, BlackBerry 10 browser, Opera Mac and Android, Internet Explorer 10 Windows Desktop, RT and Phone.

Layer

It's the base of the entire framework. Many of the other component extend it.

<div class="layer"></div>

All the code show the DOM with the classes of Heart.css, but it can be used like the Sass way.

<div id="content"></div>
#content{
    @extend .layer;
}

Topbar and Bottombar

Topbar and Bottombar respectively have a default of 44px and 50px. It can be change by redefine Sass variables : $height-topbar and $height-bottombar.

<div class="topbar"></div>
<div class="layer with-topbar with-bottombar"></div>
<div class="bottombar"></div>

Layer-Scrollable

The Layer-Scrollable is usefull if you want to create an intern scroll and disable the browser's behavior on the body's sroll.

The CSS overflow: scroll cause many problems on a mobile browser. But iScroll isn't a good solution. A CSS scroll with always be faster than a JavaScript one.

If you try this on iOS, the scroll have the native bounce and momentum effect but the body doesn't. You don't see the grey background texture of Safari anymore.

<div class="layer">
    <div class="layer-scrollable">
        <div class="wrap">
            …
        </div>
    </div>
</div>

A Layer-Scrollable always have to be use in a other Layer element.

It's the same structure than above but with less content. On iOS the content always bounce for disable the body's bounce.

Layer-Toggle

A layer-toggle is a Layer it can manage two states with a transition.

<div class="layer">
    …
</div>    
<div class="layer-toggle">
    …
</div>

Add the class .open on the .layer-toggle to open it and remove the class to close it.

<div class="layer-toggle open">
    …
</div>

It's perfect for a modal view and it's easily extandable.

Try the Demo

This example use a custom layer-toggle, it just defined an open and close state like this way :

.layer-toggle-fade{
    @extend %layer-toggle;
    opacity:0;
    &.open{
        opacity:1;
    }
}

And in this example, layer-toggle are also Layer-Scrollable

<div class="layer">
    <div class="layer-scrollable">
        <div class="wrap">
            …
        </div>
    </div>
    <div class="layer-toggle-fade layer-scrollable">
        <div class="wrap">
            …
        </div>
    </div>
</div>

Layout-Split

layout-split is a two columns layout. An a large and middle device screen the two layers appears and on a small screen only the content-layer appear, the layout-split have an state to see the aside-layer. It's a simple off-canvas layout.

<div class="layout-split">
    <div class="aside-layer">
        …
    </div>
    <div class="content-layer">
        …
    </div>
</div>

Just add the .open class to the .layout-split

<div class="layout-split open">
    …
</div>

The dimension of the layout-split can be customed with Sass variables : $split-min-width, $split-width-breakpoint and $split-width-content-extend

Try the Demo

Like the layer-toggle, you can easily define your own layout-split with your own transition.

.layout-split-carousel{
    @extend %layout-split;
    @media screen and (max-width: $split-width-breakpoint) {
        >.aside-layer{
            -webkit-transform: translateZ(-100px) rotateY(25deg);
            transform: translateZ(-100px) rotateY(25deg);
        }
        &.open{
            >.aside-layer{
                -webkit-transform: none;
                transform: none;
            }
            >.content-layer{
                -webkit-transform: translateX($split-width-aside);
                transform: translateX($split-width-aside);
            }
        }
    }
}
<div class="layout-split-carousel">
    <div class="aside-layer">
        …
    </div>
    <div class="content-layer">
        …
    </div>
</div>
